  1. start-backend.bat   (Terminal 1)
  2. start-frontend.bat  (Terminal 2)


MÉTODO 2: Manual
════════════════

BACKEND:
  cd backend
  python3 -m venv venv
  source venv/bin/activate  # Linux/Mac
  # o venv\Scripts\activate  # Windows
  pip install -r requirements.txt
  python3 init_db.py
  uvicorn app.main:app --reload --port 8000

FRONTEND:
  npm install
  npm run dev


URLS:
  Frontend:  http://localhost:5173
  Backend:   http://localhost:8000
  API Docs:  http://localhost:8000/docs

│  ⚠️  NOTAS IMPORTANTES                                              │
└──────────────────────────────────────────────────────────────────────┘

• Los scripts .sh requieren permisos de ejecución en Linux/Mac
• Los scripts .bat funcionan directamente en Windows
• El backend debe estar corriendo ANTES de ejecutar test-backend.py
• La primera ejecución instala dependencias (puede tardar)
• El puerto 8000 debe estar libre para el backend
• El puerto 5173 debe estar libre para el frontend
• Usar Python 3.8 o superior
• Usar Node.js 18 o superior


┌──────────────────────────────────────────────────────────────────────┐
│  🐛 SOLUCIÓN DE PROBLEMAS                                           │
└──────────────────────────────────────────────────────────────────────┘

"Puerto en uso":
  $ lsof -i :8000  # Ver qué usa el puerto
  $ kill -9 <PID>  # Matar proceso

"Comando no encontrado":
  • Verificar que Node.js, Python, npm y pip estén instalados
  • Verificar que estén en el PATH

"Errores de importación Python":
  $ cd backend
  $ source venv/bin/activate
  $ pip install -r requirements.txt

"Errores npm":
  $ rm -rf node_modules package-lock.json
  $ npm install
