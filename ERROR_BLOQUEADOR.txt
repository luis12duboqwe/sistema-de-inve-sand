📌 PROBLEMA IDENTIFICADO
═══════════════════════════════════════════════════════════

Tu bloqueador de anuncios (uBlock Origin, Adblock Plus, etc.) 
está bloqueando las peticiones a:
  • http://localhost:8000/api/locations/
  • http://localhost:8000/api/sales-profiles/

Esto impide que carguen las estadísticas por ubicación.


✅ SOLUCIÓN RÁPIDA (2 minutos)
═══════════════════════════════════════════════════════════

PASO 1: Identifica tu bloqueador
  → Busca el ícono en la barra de herramientas del navegador
  → Común: uBlock Origin 🛡️, Adblock Plus 🚫

PASO 2: Desactívalo para localhost
  → Haz clic en el ícono del bloqueador
  → Selecciona "Desactivar en este sitio"
  → O "Pausar en localhost"

PASO 3: Recarga la página
  → Presiona F5
  → O Ctrl + R


📋 SOLUCIÓN DETALLADA POR BLOQUEADOR
═══════════════════════════════════════════════════════════

🛡️ uBlock Origin:
   1. Clic en el ícono de uBlock
   2. Clic en el botón grande de apagado (power button)
   3. Recarga (F5)
   
   O para lista blanca permanente:
   1. Clic en el ícono de uBlock
   2. Clic en el engranaje ⚙️
   3. Pestaña "Lista blanca"
   4. Agregar: localhost:5173
   5. Aplicar cambios

🚫 Adblock Plus:
   1. Clic en el ícono de Adblock
   2. "Pausar Adblock Plus"
   3. Recarga (F5)

🔒 Brave Browser:
   1. Clic en el ícono del león (Brave Shields)
   2. Desactivar "Shields"
   3. Recarga (F5)


🔧 ALTERNATIVA: Modo Incógnito
═══════════════════════════════════════════════════════════

Los bloqueadores suelen estar desactivados en modo incógnito:

Chrome/Edge:     Ctrl + Shift + N
Firefox:         Ctrl + Shift + P
Safari:          Cmd + Shift + N (Mac)

Luego abre:      http://localhost:5173


✨ VERIFICACIÓN
═══════════════════════════════════════════════════════════

Después de desactivar el bloqueador, deberías ver:

✅ Sin advertencia amarilla en el dashboard
✅ Pestañas "Por Ubicación" y "Perfiles de Venta" funcionando
✅ Gráficos de estadísticas cargando correctamente
✅ Sin errores en la consola (F12)


🐛 SI AÚN NO FUNCIONA
═══════════════════════════════════════════════════════════

1. Verifica que el backend esté corriendo:
   → Abre: http://localhost:8000/docs
   → Deberías ver la documentación de la API

2. Si la documentación NO carga:
   → El backend no está corriendo
   → Ejecuta: ./start-backend.sh

3. Verifica la consola del navegador:
   → Presiona F12
   → Pestaña "Console"
   → Busca otros errores

💡 NOTA IMPORTANTE
═══════════════════════════════════════════════════════════

Este error SOLO ocurre en desarrollo local porque:
  • Los bloqueadores filtran URLs con "api", "ad", "track"
  • localhost no es un dominio de confianza por defecto
  • En producción no tendrás este problema

Es seguro desactivar el bloqueador para localhost.
